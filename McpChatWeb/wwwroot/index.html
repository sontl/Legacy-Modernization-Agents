📊 SQLite Database Access

          

            
Location: Data/migration.db

            
Purpose: Stores migration metadata, COBOL files, analyses, Java code

          

          
          
Common Queries

          

            
-- List all migration runs
SELECT id, status, started_at, completed_at, 
       total_files, successful_conversions 
FROM migration_runs 
ORDER BY id DESC;

-- Get files for specific run
SELECT file_name, file_type, file_path 
FROM cobol_files 
WHERE migration_run_id = 43;

-- Get analyses for specific run
SELECT cobol_file_id, analysis_json 
FROM analyses 
WHERE migration_run_id = 43;

-- Get generated Java for specific run
SELECT file_name, java_code, target_path 
FROM java_files 
WHERE migration_run_id = 43;

🔗 Neo4j Graph Database Access

          

            
Location: bolt://localhost:7687

            
Browser: http://localhost:7474

            
Credentials: neo4j / cobol-migration-2025

            
Purpose: Stores dependency graphs and file relationships

          

          
          
Common Cypher Queries

          

            
// List all runs
MATCH (r:Run) 
RETURN r.runId, r.status, r.totalFiles, r.startedAt 
ORDER BY r.runId DESC;

// Get all files for specific run
MATCH (r:Run {runId: 43})-[:CONTAINS]->(f:CobolFile) 
RETURN f.fileName, f.fileType;

// Get dependencies for specific run
MATCH (r:Run {runId: 43})-[:CONTAINS]->(source:CobolFile)
      -[d:DEPENDS_ON]->(target:CobolFile) 
RETURN source.fileName, target.fileName, d.dependencyType;

// Find circular dependencies
MATCH (r:Run {runId: 43})-[:CONTAINS]->(f:CobolFile) 
MATCH path = (f)-[:DEPENDS_ON*2..]->(f) 
RETURN [node in nodes(path) | node.fileName] as cycle;

// Get critical files (high fan-in)
MATCH (r:Run {runId: 43})-[:CONTAINS]->(f:CobolFile) 
OPTIONAL MATCH (f)<-[d:DEPENDS_ON]-() 
WITH f, count(d) as dependents 
WHERE dependents > 0 
RETURN f.fileName, dependents 
ORDER BY dependents DESC;

🌐 MCP API Access

          

            
Base URL: http://localhost:5028

            
Purpose: REST API for accessing migration data via MCP

          

          
          
Available Endpoints

          

            
# Get all run IDs
curl http://localhost:5028/api/runs/all

# Get dependencies for specific run
curl http://localhost:5028/api/runs/43/dependencies | jq '.'

# List all MCP resources
curl http://localhost:5028/api/resources

# Ask questions via chat
curl -X POST http://localhost:5028/api/chat \
  -H 'Content-Type: application/json' \
  -d '{"prompt":"Show me all dependencies for run 43"}'

# Get current run info
curl http://localhost:5028/api/runinfo

# Get data retrieval guide (this info as JSON)
curl http://localhost:5028/api/data-retrieval-guide

Quick Generate Instant Free

              

                Scans your source files with regex pattern matching to detect COBOL features
                (EXEC SQL, CICS screens, file I/O, copybooks, etc.) and builds tailored prompts from templates.
              

              
    
                
  • Detects architecture pattern (online-interactive, batch, file-processing)
    • 
                
  • Generates system + user prompts for all agents
    • 
                
  • No AI model needed — pure code analysis
    • 
                
  • Completes in under 1 second
    • 
              

              
            

          


          
          

            
🧠

            

              
AI-Enhanced Generate AI-Powered Deep Analysis

              

                Runs the same regex analysis first, then sends actual COBOL code samples
                to your selected AI model. The AI reviews your codebase and adds domain-specific
                enhancements the regex can't detect (naming conventions, business logic patterns,
                Danish-language variables, error handling idioms, etc.).
              

              
    
                
  • Everything from Quick Generate, plus:
    • 
                
  • Samples 3 largest programs + 2 largest copybooks
    • 
                
  • One AI call reviews all agents at once (~4K tokens)
    • 
                
  • AI assigns quality scores (1-10) per agent prompt
    • 
                
  • Takes 10-30 seconds depending on model speed
    •

              ℹ️
              Quality Score Explained
              
            

            

              
The Quality Score (1–10) rates the final prompt quality after AI enhancement — how ready it is for accurate code conversion.

              

                
8–10 Production-ready — comprehensive domain coverage, naming conventions, edge cases, and COBOL idioms captured.

                
5–7 Adequate — covers basics but may miss codebase-specific patterns or data structures.

                
1–4 Needs work — significant domain gaps; consider adding more source files or re-running.

              

              
How to improve scores: Ensure the source/ folder contains representative COBOL programs and copybooks. The AI samples the largest files — include your most complex programs for best results. Re-running enhancement after adding more source files will improve coverage.
